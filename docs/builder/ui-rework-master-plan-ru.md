# Builder UI Rework: Master Plan (10x10)

Дата: 2026-05-24
Статус: in-progress

## 1) Упростить правую панель (Inspector)
1. Удалить дублирующиеся секции для одного и того же блока.
2. Ввести правило: один тип блока = один основной control-блок.
3. Убрать пустые/неиспользуемые секции (`Template`, лишние `onAdd`).
4. Сделать единый порядок секций по приоритету данных.
5. Добавить краткие пояснения под заголовком каждой секции.
6. Сделать одинаковые высоты/отступы контролов.
7. Привести названия секций к единому стилю.
8. Скрывать нерелевантные секции для спец-блоков.
9. Добавить режим `advanced` по кнопке, скрытый по умолчанию.
10. Добавить smoke-check на визуальные дубли.

## 2) Data Source Mode как стандарт CMS
1. Формализовать режимы `static/element/list`.
2. Добавить mode в UI как первый шаг.
3. При смене mode синхронизировать все привязки prop.
4. Для `element` показывать выбор конкретного элемента.
5. Для `list` показывать limit/offset/sort.
6. Добавить значения по умолчанию при первом включении.
7. Покрыть edge-case пустого iblock.
8. Запретить невалидные комбинации в UI.
9. Добавить подсказки по режимам.
10. Сохранение mode в `cmsBindings`.

## 3) Отдельные инспекторы для спец-блоков
1. Выделить `SliderInspector` в отдельный файл.
2. Зафиксировать порядок полей для слайдера.
3. Добавить быстрые preset-кнопки.
4. Добавить `refresh`/reinit кнопку.
5. Убрать общие layout-секции в спец-режиме.
6. Добавить unit-проверку списка полей (snapshot).
7. Добавить поддержку будущих `HeroInspector`, `NewsInspector`.
8. Нормализовать подписи и терминологию.
9. Добавить статусы подключения прямо в спец-инспектор.
10. Убрать технические поля из основного UI.

## 4) Live status для CMS-блоков
1. Показывать состояние CMS-конфига (ok/error).
2. Показывать выбранный iblock и его label.
3. Показывать count sample элементов.
4. Показывать время последнего refresh.
5. Показывать last error в читаемом формате.
6. Добавить кнопку `recheck`.
7. Добавить предупреждение про CORS/HTTPS.
8. Добавить цветовую индикацию статуса.
9. Добавить fallback-сообщения для пустых данных.
10. Добавить журнал последних 3 операций.

## 5) Валидация mapping
1. Проверка выбранного iblock.
2. Проверка выбранного field/property для каждого prop.
3. Проверка elementId для `element` режима.
4. Проверка schema cache наличия.
5. Проверка типов поля (file/text/date).
6. Список ошибок в одной секции `Validate mapping`.
7. Быстрые действия рядом с ошибками.
8. Уровни severity (warning/error).
9. Блокировка export при критических ошибках.
10. Тест-кейсы на типовые ошибки.

## 6) Сохранение страниц
1. Кнопка `Save page now`.
2. Инлайн-статус на кнопке.
3. Авто-сброс статуса `Saved` через timeout.
4. Защита от двойного submit во время saving.
5. Prompt при закрытии вкладки с несохраненными изменениями.
6. Автосейв debounce + ручной save coexist.
7. Понятный текст ошибки при save failed.
8. Стабильная загрузка `home` по умолчанию.
9. Логика миграции старых page JSON.
10. Визуальный индикатор dirty-state.

## 7) Структура Insert/Assets/Blocks
1. Единые правила группировки компонентов.
2. Устранить дубли template в списках.
3. Устранить дубли key на React-уровне.
4. Перенести `Custom` в видимую группу.
5. Добавить сортировку по приоритету/имени.
6. Согласовать названия с manifest.
7. Добавить quick actions для блока.
8. Добавить empty-state с подсказками.
9. Улучшить поиск по alias/ключевым словам.
10. Добавить фильтры по типам блоков.

## 8) Улучшение Edit Component
1. Четко разделить page-mode и component-mode.
2. Добавить визуальный статус активного режима.
3. Убрать артефакты ресайза при переключении.
4. Нормализовать artboard outline/focus.
5. Добавить controls для колонок/ширины/gap.
6. Добавить готовые layout presets.
7. Улучшить touch/trackpad поведение.
8. Добавить hotkeys cheat-sheet.
9. Добавить reset-to-default для layout.
10. Добавить UX-тест сценарий переключений.

## 9) IDE-first workflow
1. Быстрый доступ к `preview.tsx`.
2. Быстрый доступ к `style.css`.
3. Быстрый доступ к `script.js`.
4. Показ пути текущего шаблона в UI.
5. Кнопка `copy path`.
6. Раздел “что за что отвечает” рядом с файлами.
7. Инструкция по hot reload/hard reload.
8. Подсказки про vendor assets (swiper/gsap).
9. Проверка файлов перед export.
10. FAQ для IDE-потока.

## 10) Инструкция для новичков
1. Пошаговый quick start (5 минут).
2. Сценарий “добавить блок”.
3. Сценарий “редактировать props”.
4. Сценарий “подключить CMS”.
5. Сценарий “сделать слайдер”.
6. Сценарий “редактировать код в IDE”.
7. Сценарий “сохранить страницу и проверить”.
8. Сценарий “экспортировать”.
9. Troubleshooting (CORS, пустые данные, не видно блок).
10. Визуальные примеры экранов с подписями.

## 11) Управление страницами (Pages UX)
1. Кнопка `+ New page` в секции Pages.
2. Быстрый wizard: имя + slug новой страницы.
3. Автосоздание стартового блока (Hero) для новой страницы.
4. Мгновенный переход в новую страницу после создания.
5. Отметка `Unsaved` для новой страницы до первого сохранения.
6. Валидация slug (нормализация и безопасные символы).
7. Предотвращение потери данных при переключении страниц.
8. Понятный список страниц с активной страницей и slug.
9. Быстрые действия: дублировать страницу и переименовать (следующий этап).
10. Автосохранение новой страницы в `.randee/pages/<slug>.json`.
